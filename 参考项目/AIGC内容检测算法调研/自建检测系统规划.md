# 自建AIGC检测系统规划

> 目标：搭建一个中文优化的AI生成文本检测系统
> 预计周期：3个月

---

## 一、项目目标

### 核心需求
- 检测准确率：≥90%
- 中文优化（针对知网、学术论文场景）
- 支持多模型生成文本检测（GPT-4, Claude, 文心一言, 通义千问等）
- 响应速度：<5秒/千字

### 应用场景
- 论文查重辅助
- 学术诚信检测
- 内容审核

---

## 二、技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户接口层                            │
│  (Web API / 批量检测脚本 / 浏览器插件)                       │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      预处理层                                │
│  - 文本清洗（去除引用、图表标注）                            │
│  - 分段处理（长文本分段检测）                                │
│  - 语言识别（中英文混合处理）                                │
└─────────────────────────┬───────────────────────────────────┘
                          │
          ┌───────────────┴───────────────┐
          │                               │
┌─────────▼─────────┐         ┌───────────▼───────────┐
│   零样本检测模块    │         │    特征提取模块       │
│ (Fast-DetectGPT)   │         │  (传统ML特征)         │
│                   │         │                       │
│ - 概率曲率计算     │         │ - 困惑度 (PPL)        │
│ - 条件概率采样     │         │ - 突发度 (Burstiness) │
│ - 多模型对比       │         │ - 词频分布            │
└─────────┬─────────┘         │ - 句长统计            │
          │                   │ - 文体特征            │
          │                   └───────────┬───────────┘
          │                               │
          └───────────────┬───────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                    集成分类器                                │
│  (LightGBM / XGBoost / 神经网络)                            │
│  输出：AI概率 + 置信度 + 可疑段落标注                        │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      结果输出层                              │
│  - 整体AI概率                                               │
│  - 段落级标注                                               │
│  - 可视化报告                                               │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术方案 | 理由 |
|------|----------|------|
| **零样本检测** | Fast-DetectGPT | 开源、准确率高、无需训练 |
| **大模型API** | DeepSeek / 智谱AI | 国产模型，中文支持好，成本可控 |
| **特征工程** | Python + NumPy | 成熟稳定 |
| **分类器** | LightGBM | 速度快、精度高、可解释 |
| **Web框架** | FastAPI | 异步、高性能、易部署 |
| **前端** | Streamlit | 快速搭建、适合工具型应用 |
| **部署** | Docker + Nginx | 标准化部署 |

## 三、开发路线图

### Phase 1：原型验证（第1-2周）

**目标**：跑通最小可行版本

**任务清单**：
- [ ] 环境搭建
  ```bash

  mkdir aigc-detector && cd aigc-detector
  python -m venv venv
  source venv/bin/activate
  pip install torch transformers fastapi lightgbm numpy pandas
  ```
  
- [ ] 部署 Fast-DetectGPT
  ```bash
  git clone https://github.com/baoguangsheng/fast-detect-gpt
  cd fast-detect-gpt
  # 配置模型API
  ```

- [ ] 实现基础API
  ```python
  # detector/api.py
  from fastapi import FastAPI
  
  app = FastAPI()
  
  @app.post("/detect")
  async def detect(text: str):
      result = detector.detect(text)
      return {"ai_probability": result.prob, "confidence": result.conf}
  ```

- [ ] 收集测试数据

## 五、风险与对策

| 风险 | 概率 | 对策 |
|------|------|------|
| 准确率不达标 | 中 | 多模型集成、持续迭代 |
| API成本超预算 | 中 | 缓存结果、批量处理、开源模型本地部署 |

| 知网系统更新 | 中 | 持续关注、快速适配 |
